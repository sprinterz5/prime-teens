# PrimeTeens — План платформы

> Документ-источник. Лендинг + блог сейчас, архитектура заложена под полноценную
> многоролевую платформу (родитель / студент / ментор / админ / модератор / SMM-автор).
> Код пока не пишем — это карта, по которой будем строить.

---

## 0. Что мы поняли о бренде (из гайдов)

- **Кто:** PrimeTeens помогает подросткам **13–17 лет** (и их родителям) в Казахстане
  построить **сильное портфолио** для поступления в топовые вузы мира.
- **Слоган:** «Помогаем подросткам показать лучшее в себе».
- **Аудитория двойная:** подросток (делает) + родитель (платит, принимает решение).
  Это влияет на весь копирайтинг — на лендинге должны быть два «входа».
- **Контент уже есть:** серия гайдов «Сильное портфолио» — №02 Школьный клуб,
  №03 Олимпиады, №04 Исследования / Личный проект, №05 Портфолио без бюджета,
  плюс концепция события «Хакатон / Демо-день».
- **Продуктовые направления (треки):** Олимпиады · Школьный клуб · Волонтёрство ·
  Компетишоны (хакатоны/кейсы/дебаты) · Исследовательские программы · Личный проект.
- **География / язык:** Казахстан (Астана, Алматы), языки рус / каз / англ.
- **Визуальный стиль:** тёмно-синий navy + золото/шампань, премиально, аспирационно,
  «звезда», восходящая фигура. Контраст тёмного фона и золотого акцента.

Эти гайды — готовые **лид-магниты** и **семена для блога**. Архитектуру строим так,
чтобы гайд = особый тип поста/материала (PDF + веб-версия + форма захвата email).

## 1. Принцип: «лендинг сегодня — платформа завтра»

Ключевое требование — не построить одностраничник, который потом выкинем, а заложить
фундамент платформы и **показать наружу пока только одну страницу + блог**. Поэтому:

- Один репозиторий, одно приложение, монолит на старте (не микросервисы).
- БД, роли, профили проектируем **сразу** в полном объёме, но включаем функции
  постепенно за фиче-флагами.
- Всё, что видит публика на этапе 1 — это `/` (лендинг) и `/blog`. Остальные маршруты
  (`/dashboard`, `/mentors`, `/profile`) существуют в коде, но скрыты/выключены.

## 2. Рекомендуемый стек (с обоснованием)

| Слой | Выбор | Почему |
|---|---|---|
| Фреймворк | **Next.js 15 (App Router) + TypeScript** | SSR/SSG для SEO блога, серверные экшены, один код фронт+бэк, легко растёт в платформу |
| Стили | **Tailwind CSS + shadcn/ui** | Быстрая адаптивность, дизайн-токены под бренд, готовые доступные компоненты |
| БД | **PostgreSQL** | Реляционка под роли, профили, портфолио, связи |
| Доступ к БД | **Prisma** (ORM) | Типобезопасные модели, миграции, читаемая схема |
| Auth + Storage | **Supabase** (или Auth.js + S3) | Auth с ролями, хранилище файлов портфолио, RLS, реалтайм — закрывает половину платформы из коробки |
| Контент блога | **БД-backed посты + редактор** (не голый MDX) | Роль SMM/автор требует админку без деплоя; MDX оставляем для лонгридов-гайдов |
| Хостинг | **Vercel** (фронт) + **Supabase** (БД/файлы) | Бесплатный старт, превью-деплои, масштабирование |
| i18n | **next-intl** | ru / kz / en заложить с первого дня (rkz аудитория) |
| Аналитика/формы | Plausible или GA4 + собственная таблица `leads` | Считать конверсию лендинга и захваты email |

> Альтернатива, если важнее простота и меньше кода: **Astro** для лендинга+блога
> (быстрее, статичнее), но тогда платформу с личными кабинетами придётся клеить
> отдельным приложением. **Рекомендую Next.js** именно из-за требования «расширение
> под полноценную платформу» — он один потянет и витрину, и кабинеты.

## 3. Дизайн-система (бренд-токены из логотипа)

```
Цвета
  --navy-900   #0A1A2F   (фон, основной тёмный)
  --navy-800   #0F2340
  --navy-700   #16314f
  --gold-500   #C9A24B   (основной акцент — золото)
  --gold-400   #D9B872   (ховеры, градиент)
  --champagne  #EDE3C8   (светлый текст-акцент)
  --white      #F7F8FA
  --muted      #9FB0C3   (вторичный текст)
Градиент акцента: linear-gradient(135deg, #C9A24B → #D9B872)

Типографика
  Заголовки: гротеск с характером (Unbounded / Montserrat / Manrope) — крупно, плотно
  Текст: Inter / Manrope — читабельный, нейтральный
  Поддержка кириллицы обязательна

Тон визуала
  Тёмные секции с золотым акцентом, много воздуха, звезда/«восхождение» как мотив,
  карточки со стеклянным/градиентным бордером, плавные появления при скролле.
```

Все цвета — как CSS-переменные + Tailwind theme. Тёмная тема — основная (как логотип),
светлую можно добавить позже.

## 4. Структура лендинга (одна страница, секции)

Копирайтинг берём прямо из гайдов — он уже сильный. Двойной адресат: подросток + родитель.

1. **Hero** — слоган «Покажи лучшее в себе», подзаголовок про поступление в топ-вузы,
   2 CTA: «Родителям» / «Подросткам», фон navy + золотой акцент, логотип.
2. **Проблема / миф** — «Сильное портфолио не требует денег» (из гайда №05). Цифры: 0 ₸.
3. **5 треков портфолио** — карточки: Олимпиады · Клуб · Волонтёрство · Компетишоны ·
   Исследования (иконки, краткое описание, «основатель ценнее участника»).
4. **Как мы помогаем** — гайды, события (хакатоны/демо-дни), менторство/сопровождение.
5. **Социальное доказательство** — кейсы (драма-клуб, EcoHackathon на 450 чел., медаль IBO).
   Пока нет реальных отзывов — ставим истории из гайдов + место под будущие отзывы.
6. **Лид-магнит** — «Забери гайд "Сильное портфолио"» → форма email → письмо с PDF.
   Это главный конвертер на этапе 1.
7. **Превью блога** — 3 последних поста (тянутся из БД).
8. **FAQ** — для родителей (цена, безопасность, формат) и подростков (с чего начать).
9. **Финальный CTA + футер** — контакты, Telegram/Instagram, языки, юр. инфо.

Всё — **mobile-first**, секции — переиспользуемые компоненты `<Section>`.

## 5. Блог (этап 1, но по-взрослому)

- Маршруты: `/blog` (лента + фильтр по тегам/трекам), `/blog/[slug]` (пост),
  `/blog/tag/[tag]`, RSS `/blog/rss.xml`, sitemap.
- Пост — запись в БД (см. модель ниже), а не файл, потому что роль **SMM/автор**
  должна публиковать без программиста. Черновик → ревью модератором → публикация.
- Контент-поле: rich-text (Tiptap) или MDX-в-БД. Картинки в Storage.
- SEO: метатеги, OpenGraph, JSON-LD Article, ЧПУ-слаги, канонические ссылки.
- Категории = треки портфолио (олимпиады, клубы, исследования…) — связь с продуктом.
- Гайды = посты с типом `guide` + прикреплённый PDF + форма захвата.

## 6. Модель данных (проектируем сразу под всё)

```
User
  id, email, passwordHash/oauth, role (enum), status, locale, createdAt
  Роли: STUDENT, PARENT, MENTOR, SMM_WRITER, MODERATOR, ADMIN

Profile (1:1 с User, расширяемый)
  userId, displayName, avatar, bio, city, grade(класс), school,
  interests[], languages[], socialLinks{}, visibility

StudentProfile (расширение для студента)
  userId, parentId?(связь родитель-ребёнок), portfolioId, goals, targetUniversities[]

MentorProfile (расширение для ментора)
  userId, expertise[], subjects[], hourlyInfo, verified(bool), rating

ParentLink
  parentUserId, studentUserId, relation, confirmed

Portfolio (ядро ценности платформы)
  id, studentId, summary, visibility(public/link/private)
  PortfolioItem: id, portfolioId, track(enum: olympiad/club/volunteer/
       competition/research/project), title, role(Founder/Participant…),
       description, metrics{участники, выезды, этапы}, dateFrom/dateTo,
       evidence[](файлы/ссылки/фото), level(school→intl), status

Post (блог + гайды)
  id, slug, type(article/guide/news), title, excerpt, contentRich,
  coverImage, attachments[](PDF гайдов), authorId, status(draft/review/published),
  publishedAt, tags[], track(категория), locale, seo{}, reviewerId

Lead (захваты с лендинга)
  id, email, source, guideRequested, locale, createdAt, consent

Event (хакатоны/демо-дни — на будущее)
  id, title, type, date, location, capacity, registrations[]

Comment / Reaction (на будущее, под посты и портфолио)
AuditLog (кто что изменил — нужно для модерации)
```

Все таблицы с `createdAt/updatedAt`, мягкое удаление (`deletedAt`), `locale` где нужен i18n.

## 9. Дорожная карта по этапам

**Этап 1 — Витрина (MVP, наружу)**
- Next.js + Tailwind + бренд-токены, базовый layout, тёмная тема.
- Лендинг (все секции §4), max адаптивность, lighthouse-оптимизация.
- Блог: лента + пост, посты в БД, 3–5 стартовых статей из гайдов.
- Лид-форма → таблица `leads` → авто-письмо с PDF-гайдом.
- i18n-каркас (ru сейчас, kk/en — заглушки). SEO, OG, sitemap, RSS.
- Деплой на Vercel + Supabase.

**Этап 1.5 — Студия контента**
- Auth (минимум для команды). Роли SMM_WRITER / MODERATOR / ADMIN.
- Админка постов: черновик → ревью → публикация. Загрузка картинок/PDF.
- AuditLog, базовая модерация.

**Этап 2 — Аккаунты и профили**
- Регистрация студент/родитель, связь родитель↔ребёнок.
- Профиль + публичная страница профиля.
- Портфолио: добавление айтемов по 5 трекам, метрики, доказательства, приватность.

**Этап 3 — Менторы и события**
- Профили менторов, верификация, каталог, заявки/отклики.
- События (хакатоны, демо-дни): анонс, регистрация, страница события.
- Уведомления (email/Telegram).

**Этап 4 — Полноценная платформа**
- Лента/сообщество, комментарии, реакции, рекомендации треков.
- Дашборды прогресса портфолио, шаблоны эссе/Activities list.
- Платежи/подписки, если появятся платные продукты.

## 10. Сквозные требования (не забыть)

- **Адаптивность:** mobile-first, тестировать на 360/768/1024/1440.
- **Доступность:** контраст золото-на-navy проверить (AA), фокус-стили, alt.
- **Производительность:** статическая генерация лендинга/блога, оптимизация шрифтов/картинок.
- **Безопасность:** RBAC + RLS, валидация форм (zod), защита от спама в лид-форме (honeypot/captcha),
  согласие на обработку ПДн (важно: данные несовершеннолетних → родительское согласие).
- **Юридическое:** политика конфиденциальности, согласие родителей для <18, оферта.
- **Фиче-флаги:** скрывать кабинеты/менторов, пока не готовы.
- **Контент-связь:** теги блога = треки портфолио = разделы лендинга (единая таксономия).
